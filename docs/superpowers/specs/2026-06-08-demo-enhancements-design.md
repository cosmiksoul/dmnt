## Цель

Три дополнения к существующему демо Naming Tool, все на текущих паттернах,
без бэкенда (статика, GitHub Pages, stateless):

1. **Справочник параметров** — новая read-only страница с разбивкой по уровням L0–L3.
2. **Упаковка креатива** — кнопка в Генераторе, собирающая имя креатива (каноническое + FS-safe).
3. **Планы развития** — новая read-only страница с роадмапом.

## Жёсткие ограничения (без изменений)

- Только статика, без бэкенда/БД.
- Конвенцию менять только через `src/config/convention.js`.
- UI-тесты не пишем; покрываем тестами только чистые функции в `src/lib/`.

## Фича 1 — Справочник параметров (`/reference`)

**Назначение:** единый человекочитаемый референс всех словарей конвенции, чтобы
маркетолог мог посмотреть значения кодов, не открывая Excel.

**Источник данных:** целиком из `src/config/convention.js` (`L0`, `OPTIONS`,
`L1_FIELDS`, `L2_FIELDS`, `L3_FIELDS`). Никакого дублирования данных.

**Навигация:** новый пункт меню «Справочник» в `Layout.jsx` (nav на `flex-wrap`,
6 пунктов помещаются).

**Структура страницы:**

- Якорная навигация сверху: L0 · L1 · L2 · L3 (плавный переход к секциям).
- Карточка-секция на каждый уровень (`id="L0"` … `id="L3"`).
- Внутри секции — поля уровня по порядку (как в `*_FIELDS`):
  - `kind: 'enum'` → таблица `код → расшифровка` из `OPTIONS[field.options]`
    (для L0 — из `L0`: `code → label`).
  - `kind: 'camelText'` → пометка «свободное поле — CamelCase, без `_`, пробелов, кириллицы».
  - `kind: 'date'` → пометка «дата в формате ММГГ».
- Обязательность поля (`required`) отражаем меткой (напр. звёздочка/«опц.»).

**Логика:** отсутствует. Чистая презентация data-driven из конфига → тестов нет.

- Modify: `src/components/Layout.jsx` (добавить пункт nav)

## Фича 2 — Упаковка креатива (доработка Генератора)

**Назначение:** собрать имя креатива из уровней L1–L3. По сути это имя файла
креатива для прямых медиа-размещений, но доступно для любой платформы.

**Поведение:**
- Кнопка **«Упаковать креатив»** в области результата Генератора.
- Активна, когда заполнены валидные L1–L3 (нет ошибок, есть достаточный ввод).
- По клику показывает блок с двумя значениями (каждое с кнопкой копирования):
  - **Каноническое** (по методологии): `L1|L2|L3` через `|`.
  - **Имя файла** (FS-safe): `L1-L2-L3` — одиночный `-` между уровнями.
    Внутри уровней разделитель `_`, дефисов нет → однозначно; `-` допустим
    в именах файлов на всех ОС.
- Текущий авто-блок **MEDIA** (показывался только для медиа-паблишеров)
  заменяется этим механизмом. Для медиа-паблишеров (`L0.group === 'Media'`)
  добавляем подсказку: «для прямых медиа-закупок упаковка обязательна».

**Новая чистая функция** в `src/lib/buildNames.js`:

```js
// Собирает имя креатива из L1|L2|L3.
// canonical — по методологии (через '|'); filename — FS-safe (через '-').
// Пустые уровни отфильтровываются (как в buildMediaCompound).
export function buildCreativePackage(values) {
  const parts = [buildL1(values), buildL2(values), buildL3(values)].filter(Boolean)
  return {
    canonical: parts.join('|'),
    filename: parts.join('-'),
  }
}
```

**Тесты (TDD)** в `tests/lib/buildNames.test.js`:
- Полный набор L1/L2/L3 → `canonical` через `|`, `filename` через `-`.
- Частичный набор (пустой уровень) → пустые отфильтрованы в обоих вариантах.

**Файлы:**
- Modify: `src/lib/buildNames.js` (добавить `buildCreativePackage`)
- Modify: `tests/lib/buildNames.test.js` (тесты)
- Modify: `src/pages/Generator.jsx` (кнопка + блок упаковки, замена авто-MEDIA)

## Фича 3 — Планы развития (`/roadmap`)

**Назначение:** показать на демо вектор развития — «вот куда это растёт».
Read-only, без логики.

**Навигация:** новый пункт меню «Планы развития» в `Layout.jsx`.

**Содержание (вертикальный таймлайн фаз):**

1. **Сейчас (демо):** stateless, config-driven; генерация / URL-билдер / валидатор / справочник.
2. **Этап 1 — Бэкенд + реестр:** прикрутить бэкенд, хранить утверждённые кампании.
3. **Этап 2 — Валидация против реестра:** проверка новых имён на дубли и конфликты
   с уже существующими в базе кампаниями.
4. **Этап 3 — Утверждение:** workflow внесения сгенерированной кампании в базу.
5. **Этап 4 — Авто-правила:** генерация правила проверки поступающих данных
   на основе утверждённой кампании.

**Файлы:**
